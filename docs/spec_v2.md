# Realtime Translate — VSCode 实时翻译插件 规格文档 v2

## 1. 概述

### 1.1 产品定位
一款 VSCode 插件，为阅读英文 Markdown 文件的用户提供**实时翻译**能力。用户在编辑器中选中文本后，通过快捷键或按钮触发翻译，翻译结果展示在左侧侧边栏中，并可选地在编辑器中以悬浮窗形式展示。

### 1.2 目标用户
- 使用 VSCode 阅读英文 Markdown 文档的开发者
- 习惯使用 Markdown Preview Enhanced 进行文档预览的用户

### 1.3 使用场景
1. 用户用 VSCode 打开一个英文 `.md` 文件
2. 可使用 Markdown Preview Enhanced 在侧边预览渲染效果
3. 阅读过程中遇到不理解的英文段落/句子/单词
4. 选中该文本 → 自动填充到插件侧边栏输入框 → 按快捷键或点击按钮翻译
5. 翻译结果显示在侧边栏中，可选悬浮窗同步展示

## 2. 功能需求

### 2.1 核心功能

| 编号 | 功能 | 说明 |
|------|------|------|
| F-01 | 选中文本自动捕获 | 用户在编辑器中选中文本后，自动将选中内容填充到侧边栏的待翻译输入框中 |
| F-02 | 快捷键触发翻译 | 用户按下快捷键（默认 `Ctrl+Shift+Q`）后，对输入框中的文本发起翻译请求 |
| F-03 | 按钮触发翻译 | 输入框旁的翻译按钮，点击后发起翻译请求 |
| F-04 | 翻译结果展示 | 翻译结果显示在侧边栏的只读文本框中 |
| F-05 | 悬浮窗展示（可选） | 开启悬浮窗开关后，翻译成功时额外在编辑器选中文本旁边弹出悬浮窗显示翻译结果 |
| F-06 | 翻译历史记录 | 侧边栏保留最近的翻译历史记录，数量可配置，默认 10 条 |

### 2.2 非功能性需求

| 编号 | 需求 | 说明 |
|------|------|------|
| NF-01 | 请求节流 | 避免频繁调用 API，需有防抖/节流机制 |
| NF-02 | 翻译缓存 | 相同文本不重复请求，使用本地内存缓存 |
| NF-03 | 网络异常处理 | 请求失败时给出友好提示，不阻塞后续操作 |
| NF-04 | 选中仅填充不翻译 | 选中文本后仅填充到输入框，**不自动发起翻译请求**，避免频繁调用 API |

## 3. 交互流程

### 3.1 主流程

```
用户在编辑器中选中文本
        │
        ▼
插件监听 selection 变化，自动将选中文本填充到侧边栏输入框
        │
        ▼
用户按下快捷键 Ctrl+Shift+Q（或点击翻译按钮）
        │
        ▼
插件读取输入框内容，调用 Bing 翻译 API
        │
        ├── 请求成功 ──▶ 翻译结果显示在侧边栏只读文本框中
        │                    │
        │                    ├── 悬浮窗开关开启？──▶ 在编辑器选中文本旁弹出悬浮窗
        │                    │
        │                    └── 翻译结果追加到历史记录列表
        │
        └── 请求失败 ──▶ 侧边栏显示错误提示信息
```

### 3.2 翻译历史交互

```
用户点击历史记录中的某一条
        │
        ▼
该条记录的原文和译文回填到输入框和结果文本框中
```

### 4.2 悬浮窗（仅开启开关后展示）

- 位置：编辑器中选中文本的右下方
- 样式：类似 VSCode 的 Hover 提示框，显示翻译结果文本
- 行为：翻译成功后弹出，用户点击编辑器其他位置或按 `Esc` 后消失

### 4.3 快捷键

| 快捷键 | 功能 | 生效上下文 |
|--------|------|-----------|
| `Ctrl+Shift+Q` | 触发翻译 | 编辑器中有选中文本时 |

> 快捷键可在侧边栏设置区自定义。

## 5. 翻译 API 规范

### 5.1 请求

| 项目 | 值 |
|------|-----|
| 方法 | `POST` |
| URL | `https://cn.bing.com/ttranslatev3?isVertical=1&&IG={IG}&IID=translator.5022&SFX={SFX}` |
| Content-Type | `application/x-www-form-urlencoded` |

**请求参数：**

| 参数 | 值 | 说明 |
|------|-----|------|
| `fromLang` | `auto-detect` | 源语言，固定自动检测 |
| `to` | `zh-Hans` | 目标语言，固定简体中文 |
| `tone` | `Casual` | 翻译风格 |
| `text` | `{用户选中的文本}` | 待翻译文本 |
| `token` | `{用户配置的token}` | 翻译 token |
| `key` | `{当前毫秒时间戳}` | 发起请求时的毫秒时间戳，由代码动态生成，无需用户配置 |

> `IG` 和 `SFX` 为 URL 动态参数，需从 Bing 页面获取或使用固定值。

### 5.2 响应

- Content-Type: `application/json; charset=utf-8`

**响应结构示例：**

```json
[
  {
    "translations": [
      {
        "text": "实时翻译",
        "to": "zh-Hans",
        "transliteration": {
          "text": "shíshí fānyì",
          "script": "Latn"
        }
      }
    ],
    "usedLLM": true,
    "detectedLanguage": {
      "language": "en"
    }
  }
]
```

**关键字段提取：**

| 字段路径 | 用途 |
|----------|------|
| `[0].translations[0].text` | 翻译后的中文文本 |
| `[0].detectedLanguage.language` | 检测到的源语言（可用于辅助判断） |

### 5.3 语言方向

当前固定为：**自动检测 → 简体中文（zh-Hans）**。后续根据使用体验再适配调整。

## 6. 配置项

所有配置项均可在侧边栏的设置区中修改，同时也会同步到 VSCode 的 `settings.json` 中。

| 配置键 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `realtimeTranslate.hoverEnabled` | `boolean` | `false` | 是否在编辑器中启用悬浮窗展示翻译结果 |
| `realtimeTranslate.token` | `string` | `""` | Bing 翻译 API 的 token |
| `realtimeTranslate.shortcut` | `string` | `"ctrl+shift+q"` | 触发翻译的快捷键 |
| `realtimeTranslate.historyLimit` | `number` | `10` | 翻译历史记录保留的最大条数 |

### 7.2 模块划分

| 模块 | 文件 | 职责 |
|------|------|------|
| 插件入口 | `src/extension.ts` | 注册命令、事件监听、激活插件 |
| 侧边栏 Provider | `src/providers/sidebarProvider.ts` | Webview 侧边栏的创建与消息通信 |
| 翻译服务 | `src/translation/bingService.ts` | Bing 翻译 API 请求封装 |
| 翻译缓存 | `src/translation/cache.ts` | 基于 Map 的本地翻译结果缓存 |
| 选择监听 | `src/listeners/selectionListener.ts` | 监听编辑器选中文本变化 |
| 悬浮窗 | `src/providers/hoverProvider.ts` | 编辑器中悬浮窗翻译结果展示 |
| 配置管理 | `src/config.ts` | 读取/管理插件配置 |
| 类型定义 | `src/types.ts` | 公共类型定义 |

### 7.3 消息通信（Extension ↔ Webview）

| 方向 | 消息类型 | 说明 |
|------|----------|------|
| Extension → Webview | `fillInput` | 选中文本变化，填充输入框 |
| Extension → Webview | `translationResult` | 翻译结果返回（成功/失败） |
| Extension → Webview | `updateHistory` | 更新翻译历史列表 |
| Webview → Extension | `translate` | 用户点击翻译按钮，请求翻译 |
| Webview → Extension | `configChange` | 用户修改了侧边栏配置 |
| Webview → Extension | `selectHistory` | 用户点击了某条历史记录 |

## 8. Markdown Preview Enhanced 兼容性

### 8.1 策略

| 优先级 | 方案 | 说明 |
|--------|------|------|
| P0（首版实现） | 在源码编辑器中工作 | 监听源码编辑器的 selection 变化，用户在源码侧选中文本后翻译 |
| P1（后续增强） | 在 MPE 预览窗口中工作 | MPE 预览是 Webview，需通过 Webview API 注入脚本捕获选中事件，再回传给插件 |

### 8.2 首版行为
- 插件**仅在源码编辑器中**监听选中文本
- 用户可以在左侧源码编辑器中选中文本 → 翻译
- 右侧 MPE 预览窗口保持正常渲染，不受影响

## 9. 已知限制与后续规划

### 9.1 已知限制

| 限制 | 说明 |
|------|------|
| Token 管理 | Bing 翻译 API 的 token 需要用户手动配置，当前不支持自动获取 |
| 语言方向 | 固定为自动检测→简体中文，暂不支持自定义 |
| MPE 预览窗口 | 首版不支持在预览窗口中直接选中文本翻译 |

### 9.2 后续规划

| 规划 | 说明 |
|------|------|
| 多语言支持 | 支持用户自定义目标语言 |
| MPE 预览窗口集成 | 在预览窗口中也能感知选中并翻译 |
| Token 自动获取 | 研究 Bing token 动态生成机制，实现自动获取 |
| 批量翻译 | 支持一键翻译整个文档/段落 |
